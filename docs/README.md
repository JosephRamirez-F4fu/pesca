# Project Docs

This directory centralizes the project's working documentation.

## Structure

- `backlog/`: pending work, future requests, ideas
- `plans/`: implementation plans and rollout steps
- `todos/`: active checklists and execution tracking
- `specs/`: requirements and technical definitions
- `status/`: handoffs and progress snapshots
- `archive/`: retired or superseded documents

Keep backlog, plans, TODOs, and related working documents inside this tree.
